# sml-autodiff

[![CI](https://github.com/sjqtentacles/sml-autodiff/actions/workflows/ci.yml/badge.svg)](https://github.com/sjqtentacles/sml-autodiff/actions/workflows/ci.yml)

[Automatic differentiation](https://en.wikipedia.org/wiki/Automatic_differentiation)
in pure Standard ML — compute **exact** derivatives (no finite differences) of
functions written in ordinary SML, in two complementary modes:

- **Forward mode** via [dual numbers](https://en.wikipedia.org/wiki/Dual_number)
  (`structure Dual`): a single evaluation carries a derivative alongside each
  value. Best for `deriv` (scalar `f'(x)`), directional derivatives, and small
  gradients/Jacobians seeded one coordinate at a time.
- **Reverse mode** via a [Wengert tape](https://en.wikipedia.org/wiki/Automatic_differentiation#Reverse_accumulation)
  (`structure Reverse`, type `node`): evaluate once, then a single backward
  pass yields the whole gradient of a scalar field at once — the efficient way
  to differentiate many-input functions.

The **Hessian** is computed by *forward-over-reverse*: the reverse tape carries
dual numbers, so one reverse pass per seed direction recovers the second
derivatives. No FFI, no external dependencies, and **deterministic** — every
operation is basis-library `+,-,*,/` and `Math`, so results are byte-identical
under both [MLton](http://mlton.org/) and [Poly/ML](https://www.polyml.org/).

```
=== sml-autodiff demo ========================================

1. scalar Newton for x^2 - 2 = 0  (f' via forward mode)
     start x0      = 1.0
     after 6 steps = 1.414213562373
     sqrt 2        = 1.414213562373

2. gradient of f(x,y) = (x-3)^2 + (y+1)^2 + 1/2 sin x  at (0,0)
     forward grad  = (~5.500000, 2.000000)
     reverse grad  = (~5.500000, 2.000000)
     GD step (lr = 0.1) -> (0.550000, ~0.200000)
     f(x0) = 10.000000   f(x1) = 6.903844  (decreased)

3. Newton step  x <- x - H^{-1} grad   (H via forward-over-reverse)
     Hessian row 0 = (2.000000, 0.000000)
     Hessian row 1 = (0.000000, 2.000000)
     Newton iterate = (2.750000, ~1.000000)
     f(x0) = 10.000000   f(xN) = 0.253330

==============================================================
```

*Generated by [`examples/demo.sml`](examples/demo.sml) (`make example`,
captured to [`assets/demo.txt`](assets/demo.txt)): a scalar Newton root-find
driven by forward-mode `deriv`, a 2-D gradient computed both ways (forward and
reverse agree to rounding), a gradient-descent step, and a full Newton step
using the forward-over-reverse `hessian`. All arithmetic is pure, so the output
is byte-identical run to run and across MLton and Poly/ML — note `~` is SML's
negative sign.*

## Install

With [`smlpkg`](https://github.com/diku-dk/smlpkg):

```
smlpkg add github.com/sjqtentacles/sml-autodiff
smlpkg sync
```

Include the library MLB from your own build (basis-only, no vendored deps):

```
local
  $(SML_LIB)/basis/basis.mlb
  lib/github.com/sjqtentacles/sml-autodiff/autodiff.mlb (via smlpkg)
in
  ...
end
```

This brings `structure Autodiff` into scope.

## Quick start

### Forward mode — scalar derivative

```sml
open Autodiff

(* d/dx (x^2) at x = 3  ->  6.0 *)
val s = deriv (fn x => Dual.mul (x, x)) 3.0

(* d/dx sin = cos *)
val c = deriv Dual.sin 0.7              (* ~ cos 0.7 *)
```

`deriv` takes a function written over dual numbers. You can also `open Dual` to
use the operators directly (they shadow the basis arithmetic):

```sml
fun f x = let open Dual in exp (sin x) end   (* d/dx = cos x · exp(sin x) *)
val d = Autodiff.deriv f 0.5
```

### Forward mode — gradient, directional derivative, Jacobian

```sml
open Autodiff

(* f(x,y) = x^2·y + sin x ; grad = (2xy + cos x, x^2) *)
fun f v = let open Dual
              val x = Vector.sub (v, 0)
              val y = Vector.sub (v, 1)
          in (x * x) * y + sin x end

val g  = grad f (Vector.fromList [1.3, ~0.7])             (* real vector *)
val dd = directionalDeriv f (Vector.fromList [1.3, ~0.7])
                            (Vector.fromList [0.6, ~0.8]) (* grad · dir *)

(* Jacobian of F : R^n -> R^m as a row-major real vector vector *)
fun field v = let open Dual
                  val x = Vector.sub (v, 0)
                  val y = Vector.sub (v, 1)
              in Vector.fromList [ (x * x) * y, sin x + y ] end
val J = jacobian field (Vector.fromList [1.3, ~0.7])
```

### Reverse mode — gradient in one backward pass

```sml
open Autodiff

fun f v = let open Reverse
              val x = Vector.sub (v, 0)
              val y = Vector.sub (v, 1)
          in (x * x) * y + sin x end

val g        = gradReverse f (Vector.fromList [1.3, ~0.7])
val (fx, g') = valueAndGradReverse f (Vector.fromList [1.3, ~0.7])
```

For the same function, `gradReverse` and the forward-mode `grad` agree to
rounding (the suite checks this directly).

### Hessian (forward-over-reverse)

```sml
open Autodiff

(* quadratic q(x,y) = 3x^2 + 2xy + 5y^2 ; H = [[6,2],[2,10]] (constant) *)
fun q v = let open Reverse
              val x = Vector.sub (v, 0)
              val y = Vector.sub (v, 1)
          in const 3.0 * (x * x) + const 2.0 * (x * y) + const 5.0 * (y * y) end

val H = hessian q (Vector.fromList [1.3, ~0.7])   (* row-major real vector vector *)
```

## API (`signature AUTODIFF`)

```sml
structure Dual :
sig
  type t = { v : real, d : real }
  val const : real -> t      val var : real -> t
  val value : t -> real      val deriv : t -> real
  val add : t * t -> t       val sub : t * t -> t
  val mul : t * t -> t       val divide : t * t -> t      val neg : t -> t
  val + : t * t -> t  val - : t * t -> t  val * : t * t -> t
  val / : t * t -> t  val ~ : t -> t
  val exp : t -> t   val ln : t -> t    val sin : t -> t   val cos : t -> t
  val tan : t -> t   val sqrt : t -> t  val pow : t * real -> t
end

val deriv            : (Dual.t -> Dual.t) -> real -> real
val directionalDeriv : (Dual.t vector -> Dual.t) -> real vector -> real vector -> real
val grad             : (Dual.t vector -> Dual.t) -> real vector -> real vector
val jacobian         : (Dual.t vector -> Dual.t vector) -> real vector -> real vector vector

structure Reverse :
sig
  type node
  val const : real -> node   val var : real -> node   val value : node -> real
  (* add/sub/mul/divide/neg, the + - * / ~ operators, and
     exp/ln/sin/cos/tan/sqrt/pow, all on `node` (same shapes as Dual). *)
end

val gradReverse         : (Reverse.node vector -> Reverse.node) -> real vector -> real vector
val valueAndGradReverse : (Reverse.node vector -> Reverse.node) -> real vector -> real * real vector
val hessian             : (Reverse.node vector -> Reverse.node) -> real vector -> real vector vector
```

### Conventions

- **Functions are written generically over `Dual.t` (forward) or `Reverse.node`
  (reverse).** This is how SML expresses "differentiable function" without
  type classes: you write the body once with the structure's operators and the
  library threads derivatives through it. `const` injects literals; the input
  nodes/duals are supplied by `grad`/`gradReverse`/`hessian`.
- **Vectors and matrices are plain immutable `real vector` / `real vector vector`**
  (row-major). `jacobian` returns an *m×n* matrix with `sub (sub J i, j)` =
  ∂Fᵢ/∂xⱼ; `hessian` returns the symmetric *n×n* matrix with entry *(i, j)* =
  ∂²f/∂xᵢ∂xⱼ. The library is **dependency-free**.
- **`pow` is `node/dual * real`** — a variable base raised to a constant
  exponent, `(vᶜ)′ = c·vᶜ⁻¹·v′`. General `a^b` can be written `exp (b * ln a)`.
- **Forward vs reverse.** Forward mode costs one pass per input direction, so it
  is cheapest for `f : R → R` or a few inputs / many outputs (a Jacobian
  column at a time). Reverse mode costs one backward pass for the *entire*
  gradient regardless of input count, so it wins for many-input scalar fields.

### Optional `sml-matrix` interop

The library deliberately keeps gradients/Jacobians/Hessians as plain
`real vector` / `real vector vector` so it has **no dependencies**. If you also
use [`sml-matrix`](https://github.com/sjqtentacles/sml-matrix), convert at the
boundary — e.g. build a `Matrix.t` from the Jacobian/Hessian rows via
`Matrix.fromRows` — to reuse its decompositions (`lu`, `qr`, `solve`, …) for
Newton/Gauss–Newton steps. No such dependency is required to use this library.

## Build & test

```
make test        # MLton: build + run the suite
make test-poly   # Poly/ML: use-and-run the suite
make all-tests   # both
make example     # run the gradient-descent / Newton demo, write assets/demo.txt
make clean
```

Both compilers run the same strict-TDD suite (`test/`); every comparison goes
through an explicit epsilon (never string-comparing reals):

- **Scalar derivatives** — `d/dx x² = 6` at 3 (also via `pow`), `d/dx sin = cos`
  sampled, plus `cos`, `tan`, `sqrt`, `ln`.
- **Chain rule** — `d/dx exp(sin x) = cos x · exp(sin x)` and `d/dx ln(1+x²)`.
- **Gradient** — `f(x,y) = x²y + sin x` matches the hand-derived
  `(2xy + cos x, x²)`; the directional derivative equals `grad · dir`.
- **Jacobian** — a 2-vector field against its analytic matrix.
- **Reverse vs forward** — `gradReverse` equals the forward-mode `grad` on the
  same functions, and matches a hand-derived three-variable gradient.
- **Hessian** — a quadratic's Hessian is a constant matrix (equal at different
  points), is symmetric, and matches the closed form for a non-quadratic.
